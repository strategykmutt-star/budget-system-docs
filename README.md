## เนื้อหาแต่ละส่วน

### 1. `registry/` — ระบบทะเบียน 8 ชั้น
โครงสร้างฐานข้อมูลทะเบียนงบประมาณตามที่เห็นในหน้าจอจริง (โปรแกรม `Bis65xxxx`):
กระทรวง → หน่วยงาน → เป้าหมายบริการกระทรวง/หน่วยงาน → **ผลผลิต/โครงการ** → กิจกรรม → รายการ
พร้อมตารางอ้างอิง (ปีงบประมาณ, หมวดรายจ่าย 12 หมวด, จังหวัด/พื้นที่+GPS)
- หลักการ: ผูกปีงบประมาณทุกตาราง · soft-close ก่อน hard-delete · effective dating · FK RESTRICT
- เปิด `er-diagram.html` เพื่อดูแผนภาพ · `schema.md` มี DDL ครบทุกตาราง

### 2. `kvalue/` — สูตรค่า K
ฐานข้อมูลสูตรค่า K (Escalation Factor) **35 สูตร / 5 หมวด** เก็บเป็น data model (`formulas.json`)
เพิ่ม/แก้สัมประสิทธิ์ได้โดยไม่ต้องแก้โค้ด
- สูตรหลัก `P = Po × K` · dead band 0.96–1.04 (±4%) · ยื่นภายใน 90 วัน · วงเงิน > 50 ล้านผ่าน สงป.
- เปิด `calculator.html` เพื่อคำนวณ K (เลือกประเภทงาน → กรอกดัชนี t/o → ได้ K + เงินชดเชย)
- ✅ ตรวจแล้ว: ทุกสูตรผลรวมสัมประสิทธิ์ = 1.00

### 3. `docs/` — สรุปครบทุกหน้า
สารบัญ + สรุปทีละหน้าของเอกสารอบรมทั้ง 3 ไฟล์ (174 หน้า) มุมมองพัฒนาระบบ

## ⚠️ ข้อควรระวัง (ต้องยืนยันก่อนใช้งานจริง)

- **ค่า K ถอดจากภาพสแกน (OCR)** — ผลรวมสัมประสิทธิ์ = 1.00 ทุกสูตร และตรวจซ้ำกับภาพความละเอียดสูงแล้ว แต่ควรเทียบกับต้นฉบับ
- **วิธีคิดเงินชดเชย** มี 2 แนวทาง [(K−1) เต็ม vs หักขอบ ±4%] — ต้องยืนยันกับหนังสือสำนักงบประมาณ
- **นิยามตัวแปรดัชนี** (โดยเฉพาะ M/S/E/F) ต้องยืนยันการ map กับรหัสดัชนีกระทรวงพาณิชย์ (price.moc.go.th)
- **ER schema** เป็นร่าง — มีจุดที่ต้องเคาะ (M:1 vs M:N, การ gen รหัส, audit trail) ระบุไว้ใน `schema.md`
